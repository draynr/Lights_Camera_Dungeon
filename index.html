Problem Description
Here you should provide the context for your idea. Describe the problem that you are trying to solve, why it is important, where it is challenging. Give us a general idea on how you are going to solve it.

We are going to make a top-down (2D/2.5D) bullet-hell shooter (e.g. Galaga). We plan to incorporate class concepts as mechanics in our game. One example is using lenses (optics unit) to aim the player's weapon (a laser), at the enemy. (We also have several other ideas, such as incorporating color theory into enemy bullet/projectiles). (See image below.)

We plan to use a combination of sprites, and computer-generated graphics (such as lasers/lights/glow effects).
The importance of our idea of incorporating computer graphics techniques into 2D-style games is to illustrate how even non-realistic/3D looking games can require creativity and CS184-inspired graphics techniques. Additionally, incorporating key concepts like optics diagrams as game mechanics will internalize these concepts and help educate people.

We anticipate to run into several challenges; the first is learning a game engine (we plan to use Godot, but we don't really know how to use it yet), and learning the ins and outs of 2.5D-style game development. Another challenge would be designing satisfying game mechanics that utilize course concepts (such as aiming using lens diagrams). We covered neither 2D/isometric systems, nor game design in class, so we foresee learning about these things to be time-consuming.

To solve our problem(s), we plan to use the Godot game engine (which supports both 2D and 3D games). We anticipate having to code several of the lighting/simulation effects ourselves in GDScript (or C#, which is also supported). We also anticipate having to use 3D vectors/geometry for several of our effects, although the controls will be 2D (WASD). We will have to find or create our own sprites, and we will design our own visually appealing bullet patterns, (e.g. trajectories following sinusoidal patterns, or influenced by bullet-bullet particle interactions).
